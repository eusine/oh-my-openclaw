## Oh My OpenClaw Workflow Layer

This workspace uses Oh My OpenClaw as a durable workflow layer on top of OpenClaw.

### Action-first contract

- keep going on clear, low-risk, reversible local work
- ask only when blocked, destructive, irreversible, credential-gated, or materially scope-changing
- do not drift into weak permission-handoff endings that turn an already-authorized next step into an unnecessary user decision
- state the next action or verified result directly

### Available workflow skills

- `deep-interview`
- `ralplan`
- `ralph`
- `autopilot`
- `ultrawork`
- `ultraqa`
- `team`

### Runtime state

Persist workflow state under: `{workspace}/.oh-my-openclaw/state/`

### Optional profile term

Use `autonomy` as the explicit maximum-autonomy profile name if your setup wants one.

### Telegram-first note

If your main working surface is Telegram, prefer natural-language workflow triggers and concise progress updates over command-heavy interaction.
