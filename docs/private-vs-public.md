# Private vs Public Boundary

## Keep private

- `SOUL.md`, `USER.md`, `MEMORY.md`
- `memory/`
- `.openclaw/`
- live private runtime state, plans, logs, and session artifacts
- machine-specific notes from `TOOLS.md`

## Safe to publish after review

- generalized workflow skills
- generic helper scripts
- installation notes
- sanitized examples
- migration guides

## Review manually before push

- hardcoded paths
- usernames
- hostnames
- internal repo names
- dates or notes that reveal private operating history
