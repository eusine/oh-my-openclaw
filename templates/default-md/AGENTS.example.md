# AGENTS.md

## Purpose

This workspace is home. Treat it like an operating environment, not a random folder.

## Startup

On session start:
1. read `SOUL.md`
2. read `USER.md`
3. read recent daily notes if your workspace uses them
4. read `MEMORY.md` only in private one-to-one contexts

## Workflow

- do the work before talking about the work
- finish the real fix when it is within reach
- delegate when parallel work helps
- do not expose private data in shared spaces

## Boundaries

- ask before public posting or irreversible actions
- keep machine secrets and personal notes out of version control
- prefer templates and docs over publishing filled-in private files
